# ACL 风格参考

## 用途

在根据研究笔记起草或修改 ACL 风格论文散文时使用本参考。

目标不仅是模仿表面上的润色。目标是生成清晰、技术精确、注重证据、结构严谨的散文。

## 核心风格

- 使用清晰、直接、技术精确的散文。
- 优先使用明确的主张，而非修辞华丽的辞藻。
- 将每一个强有力的主张与证据、推导、引用或清晰陈述的推理联系起来。
- 当证据不完整时，使用谨慎的模态表达。
- 在整个章节中保持术语和符号的一致性。
- 在依赖关键概念之前先定义它们。
- 优先使用具体描述，而非模糊赞扬。
- 使句子之间的逻辑关系易于理解。

## 语气

ACL 风格散文通常：

- 克制而非宣传性
- 分析性而非印象性
- 明确而非暗示性
- 具体而非笼统
- 对不确定性保持谨慎

优先使用如下措辞：

- "We study..."
- "We consider..."
- "We define..."
- "Our results suggest..."
- "These findings indicate..."
- "This is consistent with..."
- "One possible explanation is..."
- "A limitation of this analysis is..."

避免如下措辞：

- "This clearly proves..."
- "This dramatically outperforms..."
- "This is groundbreaking..."
- "This perfectly solves..."
- "This undeniably shows..."

## 主张强度

使措辞强度与笔记中的支持相匹配。

仅在笔记明确支持时使用强硬措辞。

- 仅在证据强有力且直接时使用 "shows" 或 "demonstrates"。
- 当支持是部分的、间接的或观察性的时候，使用 "suggests" 或 "indicates"。
- 当仍存在多种解释时，使用 "is consistent with"。
- 当想法合理但尚未证实时，使用 "we hypothesize" 或 "we conjecture"。

如果笔记不完整，应弱化主张而非编造支持。

## 基于笔记写作

当源材料是原始笔记堆时：

- 即使原始措辞粗糙，也要保留底层技术含义
- 润色局部措辞，但不要编造缺失的实验细节、结果、引用或定理条件
- 当翻译会模糊含义时，保留重要领域特定术语的原始形式
- 如果几个笔记片段表达相同观点，将它们合并为一个连贯的段落
- 如果笔记存在歧义，要么明确标记，要么选择最窄的可辩护表述
- 区分笔记中有据可查的内容与仅为提高可读性而引入的内容

好的草稿应感觉达到发表水准，同时忠实于笔记。

## 段落结构

一个强有力的 ACL 风格段落通常做到以下几点：

1. 陈述观点。
2. 定义对象、问题或对比。
3. 解释方法、推理或证据。
4. 用证据、推导或比较支撑主张。
5. 陈述启示、局限性或过渡。

不是每个段落都需要全部五个步骤，但大多数强有力的段落应有清晰的内部逻辑。

## 章节模式

### 问题设定

当章节介绍任务、背景或 motivating challenge 时使用。

一个好的问题设定段落通常：

1. 定义研究对象
2. 解释问题为何困难或重要
3. 指出现有方法或理解中的缺口
4. 陈述本章节更聚焦的范围

有用的技巧：

- 在评估之前先定义任务
- 明确命名困难的来源
- 区分一般问题与本文解决的特定问题

### 方法

方法章节应帮助读者重构做了什么以及为什么这样做。

一个好的方法段落通常：

1. 陈述组件或设计选择
2. 解释其在整体方法中的作用
3. 提供足够细节以理解机制
4. 解释该选择的合理性

有用的技巧：

- 在使用符号前先引入符号
- 明确解释输入、输出和转换
- 将设计动机与实现细节分开
- 尽可能每个概念组件单独成段

方法段落模板：

- "We model X as Y."
- "Given A, the system computes B by ..."
- "This design allows ..."
- "In contrast to prior approaches, this component ..."

### 相关工作

相关工作章节应以分析方式组织前人工作，而非简单罗列论文。

一个好的相关工作段落通常：

1. 命名一条研究脉络或比较维度
2. 按共同方法、假设或目标对前人工作进行分组
3. 陈述这些工作的成果
4. 指出与本文相关的具体区别

有用的技巧：

- 沿清晰维度比较前人工作
- 明确陈述与当前论文的关系
- 避免没有比较依据的模糊新颖性主张

相关工作段落模板：

- "Prior work on X has mainly followed two directions."
- "The first line focuses on ..., whereas the second emphasizes ..."
- "Our setting differs in that ..."
- "This distinction matters because ..."

### 分析

分析章节应解释结果，而非简单重复。

一个好的分析段落通常：

1. 陈述模式或发现
2. 解释其重要性
3. 提供一个技术上合理的解释
4. 在需要时指出不确定性、范围或替代解释

有用的技巧：

- 将解释与方法或任务联系起来
- 区分观察与解释
- 除非有依据，否则避免因果语言
- 注意解释是试探性的

分析段落模板：

- "We observe that ..."
- "One plausible explanation is ..."
- "This pattern is consistent with ..."
- "However, the evidence does not yet determine whether ..."

### 局限性

局限性章节应指出真实约束，而非程式化的敷衍。

一个好的局限性段落通常：

1. 指出具体局限性
2. 解释其来源
3. 说明它如何影响解释或泛化
4. 对未来工作提出有边界的启示

有用的技巧：

- 具体说明范围条件
- 区分数据局限性、建模局限性和评估局限性
- 解释不能得出什么结论

## 句子层面指导

优先：

- 当内容密集时，每句一个技术要点
- 使用明确连接词，如 "because"、"therefore"、"however"、"in contrast"、"specifically"
- 引入术语时使用简短的同位语说明
- 使用具体名词而非模糊抽象概念

注意：

- 包含多个未标记逻辑转换的长句
- 指代不清的代词
- 笔记中未解释的行话
- 符号过载
- 未经重构直接粘贴成散文的压缩要点

## 应避免什么

- 没有比较依据的新颖性主张
- 没有证据的模糊短语，如 "very effective"
- 从相关观察中得出因果主张
- 先使用后引入的符号
-  unsupported universals，如 "always"、"in general"、"all"
- 超出证据的夸大 framing
- 假设尚未定义概念的段落开头
- 没有分析线索的观察列表
- 听起来润色但掩盖缺失技术内容的散文

## 最终起草规则

草稿应比笔记更流畅，但绝不能比笔记所证明的更确定。
