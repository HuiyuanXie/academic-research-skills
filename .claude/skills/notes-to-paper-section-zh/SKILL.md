# 笔记转论文章节

## 用途

阅读原始研究笔记，理解技术内容，识别逻辑或技术问题，并起草一篇润色后的 ACL 风格章节。

## 工作流

1. 在起草前完整阅读笔记。
2. 在内部构建关于主张、假设、证据、定义和开放性缺口的地图。
3. 写作前，阅读 [references/critique-checklist.md](references/critique-checklist.md)，并用其检验技术正确性和逻辑严谨性。
4. 向用户报告技术疑虑、逻辑缺口和缺失信息。
5. 起草最终散文时，阅读 [references/acl-style.md](references/acl-style.md)，并遵循其在语气、段落结构和主张约束方面的指导。
6. 清楚区分笔记中有据可查的内容与为可读性而推断的内容。
7. 绝不编造笔记未支持的结果、引用、数据集或定理条件。

## 输出格式

返回：

1. 对笔记的理解
2. 技术和逻辑批判
3. 缺失信息
4. 改进建议
5. ACL 风格章节草稿
6. 假设与不确定性
