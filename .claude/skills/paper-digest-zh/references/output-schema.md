# 输出模式

## 用途

在对论文分类后，使用此模式构建摘要。

返回带有清晰标题和子标题的 Markdown。

## 通用部分

对所有论文类型包含以下部分：

### 标题

- 论文完整标题

### 一句话总结

- 一句话概括论文主要贡献

### 论文类型与关键词

- 主要论文类型
- 相关技术关键词

### 作者陈述的贡献

- 尽可能直接提取自论文

## 基准或数据集论文

如果论文分类为 `benchmark/dataset`，包含：

### 数据集名称

### 数据集亮点

- 关键创新点
- 区分性特征

### 子任务 / 数据子集

### 示例样本

### 数据统计

- 示例数
- 类别数
- 划分
- 规模
- 论文中给出的其他关键统计

### 原始数据来源

### 构建过程

- 合成、自动、人工标注或混合

### 实验设计与结果总结

- 实验中使用的数据集或任务
- 基线或比较系统
- 评估指标
- 实验设置
- 主要定量结果
- 原始结果表格或图表，忠实重现或描述

### 分析 / 讨论

- 论文自己的见解和讨论点

## 建模论文

如果论文分类为 `modelling`，包含：

### 方法一句话总结

### 方法亮点

- 新颖之处
- 主要设计贡献

### 模块 / 步骤

- 将方法拆分为模块或顺序步骤
- 对每个模块或步骤给出简要总结

### 决策点与假设

- 重要设计选择
- 假设
- 约束

### 实验设计与结果总结

- 使用的数据集或任务
- 基线或比较系统
- 评估指标
- 当训练或推理设置是论文核心时的相关细节
- 主要定量结果
- 原始结果表格或图表，忠实重现或描述

### 分析 / 讨论

- 分析
- 消融
- 讨论

## 综述论文

如果论文分类为 `survey`，包含：

### 综述逻辑与结构

### 分节总结

- 总结每个主要章节

### 高影响力引用

对每个重点引用，包含：

- 标题
- 一句话总结
- 链接（如果源内容中可获取）

## 立场论文

如果论文分类为 `position`，包含：

### 核心立场

### 论证流程

### 对立立场

### 反驳

- 解释作者为何拒绝或驳斥替代立场

### 分析

- 严格基于论文自身内容

## 完整性规则

- 如果某个请求部分无法从论文中可靠填充，请保留标题并简要说明信息未明确说明或无法从可用来源获取。
- 当模式期望某个部分时，不要默默省略它。

## 格式规则

- 保持标题明确
- 提取内容为列表形式时，使用简洁要点
- 可读时以 Markdown 形式保留表格
- 否则忠实且具体地描述图表
- 所有总结均基于论文内容
