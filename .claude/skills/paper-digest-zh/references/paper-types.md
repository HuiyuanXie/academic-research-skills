# 论文类型

## 用途

在撰写摘要前使用本参考对论文进行分类。

根据论文的核心贡献选择主导论文类型，而非孤立的章节。

## 1. 基准或数据集论文

当主要贡献为以下一项或多项时，将论文分类为 `benchmark/dataset`：

- 引入新数据集
- 引入新基准或评估套件
- 以数据整理、标注或构建为核心贡献
- 围绕数据集或基准定义子任务、划分或评估设置

常见信号：

- 标题突出数据集或基准名称
- 论文大部分内容聚焦于数据收集、标注或统计
- 实验围绕在新资源上基准测试模型展开

即使论文包含基线模型，只要数据集或基准是主要贡献，就保留此标签。

## 2. 建模论文

当主要贡献为以下一项或多项时，将论文分类为 `modelling`：

- 新模型、架构、训练方法、推理方法或算法
- 更大系统中的新模块或设计模式
- 主要证据为经验或理论性能的方法改进

常见信号：

- 标题突出模型或方法
- 论文包含围绕组件或步骤组织的方法章节
- 实验主要用于验证所提出的方法

如果论文同时引入数据集和方法，仅当方法显然是主要贡献时才选择 `modelling`。

## 3. 综述论文

当主要贡献是综合而非新模型或数据集时，将论文分类为 `survey`。

常见信号：

- 论文将前人工作按主题、分类法或历史发展进行组织
- 论文比较研究脉络而非提出新系统
- 论文的价值在于覆盖范围、结构、批判或综合

摘要应重点关注综述的组织方式及其在各章节中得出的结论。

## 4. 立场论文

当中心目标是论证某种观点、框架、议程或规范性立场时，将论文分类为 `position`。

常见信号：

- 论文主张该领域应该做什么、避免什么、重新定义什么或优先什么
- 贡献是一种立场、论点或概念框架，而非新系统
- 论文讨论替代方案并论证反对它们

摘要应强调论证结构和反驳，而不是将论文呈现为经验结果论文。

## 分类规则

- 为摘要选择一个主要类型。
- 如果论文是混合类型，选择最符合作者声称的主要贡献的类型。
- 仅在有用时提及显著的次要特征，但保持输出模式锚定在主要类型上。
- 如果不确定，简要说明不确定性，并选择最窄的可辩护标签。
